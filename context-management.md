# Context management

## Never throw away good context
**Source:** Matt Pocock video — https://youtu.be/UzMNBN6xLLA?si=_cJ76bfFtPqM7hRE&t=418

A useful idea from Matt's video: never throw away good context.

If you discover a prompt, workflow, explanation, checklist, or chunk of reasoning that works well, save it in a document so you can reuse it later instead of recreating it from scratch.

Good context to preserve:
- effective prompts
- debugging checklists
- reusable implementation plans
- code review heuristics
- architecture notes
- examples that produced strong results

Treat useful context like a reusable asset. If it helped once, it may help again.
